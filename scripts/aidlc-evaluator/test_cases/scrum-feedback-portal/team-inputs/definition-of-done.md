# PulseBoard — Definition of Done (Team-Owned)

An increment is **Done** only when every applicable criterion is met (enforced at
Sprint Review):

- [ ] All committed stories' acceptance criteria are met with tested evidence.
- [ ] Code adheres to `coding-conventions.md` and fits `architecture.md`.
- [ ] Unit tests pass; ≥80% coverage on internal code paths.
- [ ] Contract tests pass for any REST endpoints added/changed.
- [ ] Lint and type-check pass with zero errors/warnings.
- [ ] Any `research` spike in the increment has an approved recommendation, and
      the design/code reflect it.
- [ ] No secrets committed; inputs validated at boundaries.
- [ ] Docs/README updated for new endpoints or setup steps.
